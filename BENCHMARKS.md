### How to read these

- **Load**: `whisper.load_model(name)` wall-clock. The first time a
  model is requested, this includes the download from
  `openaipublic.azureedge.net` (which is what makes `large-v3-turbo`
  look slow at 116 s — that's a 1.5 GB download). Subsequent loads
  are pure disk reads.
- **Warm-up**: first `transcribe()` call. PyTorch lazily compiles
  kernels on the first run, so it's slower than the timed runs that
  follow.
- **Median run**: median of three timed transcriptions of the same
  29.4 s clip. The clip is fixed; only the model varies.
- **Realtime factor**: `clip_duration / median_run`. Above 1.0 = faster
  than real-time. The streaming-preview popup wants this comfortably
  above 1.0 to feel live.
- **WER**: lower is better. 0% = perfect, 100% = every word wrong.
  Differences below ~3 percentage points on a single 68-word clip are
  not statistically significant — re-run with `--runs 5` and a
  longer clip if you need finer resolution.

## Reproducing this run

```bash
# from the repo root
python benchmark.py samples/librispeech-1272-128104-0004.flac \
    --reference samples/librispeech-1272-128104-0004.txt \
    --models tiny.en base.en small.en medium.en medium large-v3-turbo \
    --runs 3
```

The script writes `benchmark_results.json` (machine-readable) and
`BENCHMARKS.md` (this file) next to itself. Pre-warm the model cache
first if you don't want the first-launch download to dominate the
`load_time` numbers — see [docs/MODELS.md](docs/MODELS.md).

## What this clip is and why it's a fair benchmark

The clip is utterance 0004 of chapter 128104 in the LibriSpeech
`dev-clean` corpus — a 29-second excerpt from a 19th-century essay
read by speaker 1272. We picked it because:

- The reference text is verified by the LibriSpeech maintainers, not
  generated by another model.
- It's long enough that load and warm-up costs don't dominate the
  numbers (the previous JFK clip was 11 s, where `medium`'s warm-up
  alone was longer than its run).
- It contains proper nouns (Linnell, Birket Foster, Carker, Mason),
  archaic idioms ("up guards and at em"), and rhetorical structure —
  the kinds of inputs where the bigger models earn their cost over
  the smaller ones.
- It's CC BY 4.0 and ships in `samples/`, so anyone running the
  benchmark gets identical input.
